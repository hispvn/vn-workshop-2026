# Measure and MeasureReport

Health information systems like DHIS2 are built around two fundamentally different kinds of data. On one side, there are individual clinical observations -- a patient's weight, a lab result, a diagnosis. On the other, there are aggregate metrics and calculated indicators -- "150 children received DPT3 this quarter at District Hospital" or "malaria incidence rate is 12.3 per 1,000 population." FHIR handles the first kind well with Observation, Condition, and other clinical resources. But for the second kind -- the summarized, counted, and calculated values that drive public health decision-making -- FHIR provides two dedicated resources: **Measure** and **MeasureReport**.

A Measure is a computable definition. It describes *what* should be calculated: the populations to include or exclude, the numerator and denominator of a rate, or the criteria for counting events. Think of it as a recipe. A MeasureReport is the result of applying that recipe to real data: the actual numbers, scores, and counts for a specific facility, district, or time period. This separation between definition and result is deliberate. A single Measure definition for "ANC Coverage" can produce thousands of MeasureReports -- one for every facility and every reporting period in the system.

This distinction matters enormously for the DHIS2 use case. DHIS2's aggregate data model is built on exactly this pattern. A **Data Set** defines which data elements a facility must report each month. An **Indicator** defines how to calculate a rate or ratio from those data elements. The actual reported numbers live in **Data Value Sets**, and the calculated results appear in the analytics engine. Measure maps to the definition side (Data Sets and Indicators), while MeasureReport maps to the data side (Data Value Sets and analytics output). This chapter explores both resources in detail, shows how they connect to DHIS2 concepts, and provides practical profiles and examples for building the bridge between the two systems.

Understanding Measure and MeasureReport is arguably the most important part of a DHIS2-to-FHIR mapping. While tracker data maps relatively cleanly to Patient, Encounter, and Observation, aggregate data has no natural home in FHIR without these resources. If your goal is to exchange routine facility reports, disease surveillance summaries, or indicator values between DHIS2 and other systems using FHIR, this chapter is where the core patterns live.
