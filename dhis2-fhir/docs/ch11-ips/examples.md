# Examples

The IG includes **55 complete IPS documents**, each containing a Patient, 3–7 Immunizations, a Composition, and a Bundle. These are auto-generated by `gen_examples.py` using a deterministic seed (`random.seed(99)`) for reproducible output.

## Example Statistics

| Resource | Count | Notes |
|----------|-------|-------|
| DHIS2IPSPatient | 55 | Diverse demographics across 11 countries |
| DHIS2IPSImmunization | 265 | 3–7 per patient, 25 different vaccines |
| DHIS2IPSComposition | 55 | One per patient |
| DHIS2IPSBundle | 55 | One complete IPS document per patient |
| **Total instances** | **430** | |

Each patient has:
- A unique DHIS2 UID (11-character alphanumeric)
- 70% have a national identifier
- Realistic given and family names
- Birth dates between 1955 and 2020
- Approximately 50/50 male/female split

## Vaccine Coverage

The 265 immunizations use 25 different CVX codes spanning:

**EPI Childhood Vaccines:**
BCG, Hep B, OPV, IPV, DTaP, Hib, PCV13, Rotavirus, MMR, Varicella, Hep A, Yellow Fever, HPV, Meningococcal A

**Adult/Travel Vaccines:**
Tdap, Td, Pneumococcal polysaccharide, Influenza, Rabies, Typhoid oral

**COVID-19 Vaccines:**
Pfizer (CVX 208), Moderna (CVX 207), AstraZeneca (CVX 212), Sinovac (CVX 511)

Each immunization includes:
- CVX-coded vaccine type
- Administration date (2018–2025)
- Lot number (e.g., `AB1234`, `GH5678`)
- Route (intramuscular, subcutaneous, or oral)
- Injection site for non-oral vaccines (left/right upper arm or thigh)
- Dose number
- Target disease (SNOMED CT coded)

## Walkthrough: A Complete IPS Document

Let's trace through the first generated IPS document to see how all four resources connect.

### The Patient (IPSPatient001)

```fsh
Instance: IPSPatient001
InstanceOf: DHIS2IPSPatient
* id = "ips-patient-001"
* identifier[dhis2uid].value = "rNmK4jPfiOd"    // DHIS2-generated UID
* name[0].family = "Camara"
* name[0].given[0] = "Quynh"
* gender = #female
* birthDate = "1999-06-07"
* address[0].city = "Kumasi"
* address[0].country = "GH"
```

### An Immunization (one of several)

```fsh
Instance: IPSImm00101
InstanceOf: DHIS2IPSImmunization
* id = "ips-imm-001-01"
* status = #completed
* vaccineCode.coding[cvx].system = "http://hl7.org/fhir/sid/cvx"
* vaccineCode.coding[cvx].code = #19
* vaccineCode.coding[cvx].display = "BCG"
* patient = Reference(IPSPatient001)
* occurrenceDateTime = "2023-08-14"
* lotNumber = "GH3847"
* site.coding[0] = http://snomed.info/sct#368208006 "Left upper arm"
* route.coding[0] = http://snomed.info/sct#78421000 "Intramuscular route"
* protocolApplied[0].doseNumberPositiveInt = 1
* protocolApplied[0].targetDisease[0].coding[0]
    = http://snomed.info/sct#56717001 "Tuberculosis"
```

### The Composition

```fsh
Instance: IPSComposition001
InstanceOf: DHIS2IPSComposition
* id = "ips-composition-001"
* status = #final
* type = http://loinc.org#60591-5 "Patient summary Document"
* subject = Reference(IPSPatient001)
* date = "2025-01-15"
* author[0].display = "DHIS2 System"
* title = "International Patient Summary"
* section[immunizations].title = "Immunizations"
* section[immunizations].code = http://loinc.org#11369-6
* section[immunizations].entry[0] = Reference(IPSImm00101)
* section[immunizations].entry[1] = Reference(IPSImm00102)
* section[immunizations].entry[2] = Reference(IPSImm00103)
// ... more entries for all immunizations
```

### The Bundle

```fsh
Instance: IPSBundle001
InstanceOf: DHIS2IPSBundle
* id = "ips-bundle-001"
* type = #document
* timestamp = "2025-01-15T10:00:00Z"
* entry[0].fullUrl = "urn:uuid:ips-bundle-001-comp"
* entry[0].resource = IPSComposition001            // Composition first
* entry[1].fullUrl = "urn:uuid:ips-patient-001"
* entry[1].resource = IPSPatient001                // Patient second
* entry[2].fullUrl = "urn:uuid:ipsimm00101"
* entry[2].resource = IPSImm00101                  // Immunization 1
* entry[3].fullUrl = "urn:uuid:ipsimm00102"
* entry[3].resource = IPSImm00102                  // Immunization 2
// ... remaining immunizations
```

The Bundle is the **complete, self-contained IPS document**. A receiving system can process this single JSON payload and reconstruct the full patient summary.

## Regenerating Examples

The examples are generated by a Python script using deterministic randomization:

```bash
make ips-examples
# or: python3 ig/input/fsh/ips/gen_examples.py
```

To change the number of patients or vaccine distribution, edit `gen_examples.py` and rerun. The output is written to `ig/input/fsh/ips/examples-generated.fsh`.

After regenerating, recompile with SUSHI:

```bash
make docker-sushi
```

## Viewing in the Renderer

The IPS examples are compiled by SUSHI into JSON and are visible in the local renderer:

- **Patient list**: All 55 IPS patients appear alongside other patients at `/patients`
- **Questionnaire responses**: IPS Bundles appear in the QuestionnaireResponse list (as they are resource instances)
- **JSON view**: Each resource is viewable as raw JSON at `/json/{type}/{id}`
