# DHIS2Patient Profile

The `DHIS2Patient` profile maps a DHIS2 Tracked Entity Instance (TEI) of type Person to a FHIR Patient resource. It is the most fundamental profile in the IG -- almost every tracker-based workflow starts with a patient.

## DHIS2 Context

In DHIS2, patients are modeled as **Tracked Entity Instances (TEIs)** of type "Person." Each TEI has:

- A unique auto-generated **UID** (11 alphanumeric characters, e.g., `DXz2k5eGbri`)
- An **owning Organisation Unit** (the facility where the person was registered)
- **Tracked Entity Attributes** (TEAs) for demographics: name, gender, date of birth, national ID, address, phone number, etc.

The profile ensures that every FHIR Patient resource created from a DHIS2 TEI carries enough information for round-trip interoperability.

## Identifier Slicing

The profile uses **identifier slicing** with a `#pattern` discriminator on `type` to define two well-known identifier slots. Slicing allows the profile to require specific identifiers while still permitting additional ones.

### Slice: `dhis2uid` (1..1 MS)

The DHIS2 TEI UID -- always required, exactly one.

| Element | Constraint |
|---------|------------|
| `system` | Fixed to `http://dhis2.org/fhir/id/tracked-entity` |
| `type` | Fixed to `RI` from HL7 v2 Table 0203 |
| `value` | Must satisfy `dhis2-uid-format` invariant |

The type code **`RI`** means **Resource Identifier** -- a system-generated ID assigned by the source system. This is the correct code for DHIS2 UIDs because they are auto-generated by the DHIS2 server, not assigned by an external authority.

The `dhis2-uid-format` invariant enforces the DHIS2 UID format with the FHIRPath expression:

```
matches('^[a-zA-Z][a-zA-Z0-9]{10}$')
```

This ensures exactly 11 characters, starting with a letter.

### Slice: `national` (0..1 MS)

A government-issued national identifier -- optional but Must Support.

| Element | Constraint |
|---------|------------|
| `system` | 0..1 (varies by country) |
| `type` | Fixed to `NI` from HL7 v2 Table 0203 |
| `value` | 1..1 required |

The type code **`NI`** means **National unique Individual Identifier** -- a government-issued ID such as a national ID number or social security number.

### HL7 v2 Table 0203 -- Identifier Type Codes

This table defines the standard codes for classifying identifiers. The codes used in this IG:

| Code | Meaning | Use Case |
|------|---------|----------|
| `RI` | Resource Identifier | System-generated ID (DHIS2 UIDs) |
| `NI` | National Identifier | Government-issued national ID |
| `MR` | Medical Record Number | Facility-assigned patient number |
| `PP` | Passport Number | Travel document identifier |
| `PPN` | Passport Number (alt) | Alternative passport code |
| `DL` | Driver's License | Driver's license number |

## Required Demographics

The profile mandates core demographics that align with DHIS2 Tracker's typical required attributes for person-type tracked entities:

| Element | Cardinality | Notes |
|---------|-------------|-------|
| `name` | 1..* MS | At least one name required |
| `name.family` | 1..1 | Family (last) name |
| `name.given` | 1..* | Given (first) name(s) |
| `gender` | 1..1 MS | Maps to DHIS2 sex attribute |
| `birthDate` | 1..1 MS | Obeys `dhis2-birthdate-not-future` |
| `address` | 0..* MS | Must Support but optional |

The `dhis2-birthdate-not-future` invariant prevents future birth dates:

```
$this <= today()
```

## Organisation Unit Extension

Every TEI in DHIS2 is "owned by" an Organisation Unit. The profile includes `DHIS2OrgUnitExtension` (named `orgUnit`) as 0..1 MS to carry a reference to the owning DHIS2Organization.

## FSH Source

```fsh
Profile: DHIS2Patient
Parent: Patient
Id: dhis2-patient
Title: "DHIS2 Patient"
Description: """
A Patient resource representing a DHIS2 Tracked Entity Instance (TEI) of type
Person. This profile ensures that essential demographics and the DHIS2 UID are
always present, enabling round-trip interoperability between DHIS2 Tracker and
FHIR-based systems.
"""

* identifier 1..* MS
* identifier ^slicing.discriminator.type = #pattern
* identifier ^slicing.discriminator.path = "type"
* identifier ^slicing.rules = #open
* identifier ^slicing.ordered = false

* identifier contains dhis2uid 1..1 MS
* identifier[dhis2uid].system 1..1
* identifier[dhis2uid].system = $DHIS2-TEI (exactly)
* identifier[dhis2uid].type 1..1
* identifier[dhis2uid].type = $V2-0203#RI
* identifier[dhis2uid].value 1..1
* identifier[dhis2uid].value obeys dhis2-uid-format

* identifier contains national 0..1 MS
* identifier[national].system 0..1
* identifier[national].type 1..1
* identifier[national].type = $V2-0203#NI
* identifier[national].value 1..1

* name 1..* MS
* name.family 1..1
* name.given 1..*

* gender 1..1 MS
* birthDate 1..1 MS
* birthDate obeys dhis2-birthdate-not-future

* address MS

* extension contains DHIS2OrgUnitExtension named orgUnit 0..1 MS
```

## RuleSet: DHIS2PatientIdentifiers

A reusable RuleSet for populating identifier values on DHIS2Patient instances. It reduces repetition across examples:

```fsh
RuleSet: DHIS2PatientIdentifiers(dhis2uid, natId, natSystem)
* identifier[dhis2uid].system = $DHIS2-TEI
* identifier[dhis2uid].type = $V2-0203#RI
* identifier[dhis2uid].value = "{dhis2uid}"
* identifier[national].system = "{natSystem}"
* identifier[national].type = $V2-0203#NI
* identifier[national].value = "{natId}"
```

Usage in an instance:

```fsh
* insert DHIS2PatientIdentifiers(DXz2k5eGbri, 12345678901, urn:oid:2.16.454.1)
```

## Example: PatientJaneDoe

The primary example instance -- a 35-year-old female from Lilongwe, Malawi, registered at Facility Alpha Health Center:

```json
{
  "resourceType": "Patient",
  "id": "PatientJaneDoe",
  "meta": {
    "profile": ["http://dhis2.org/fhir/StructureDefinition/dhis2-patient"]
  },
  "identifier": [
    {
      "type": {
        "coding": [
          {
            "system": "http://terminology.hl7.org/CodeSystem/v2-0203",
            "code": "RI"
          }
        ]
      },
      "system": "http://dhis2.org/fhir/id/tracked-entity",
      "value": "DXz2k5eGbri"
    },
    {
      "type": {
        "coding": [
          {
            "system": "http://terminology.hl7.org/CodeSystem/v2-0203",
            "code": "NI"
          }
        ]
      },
      "system": "urn:oid:2.16.454.1",
      "value": "12345678901"
    }
  ],
  "active": true,
  "name": [
    {
      "use": "official",
      "family": "Doe",
      "given": ["Jane"]
    }
  ],
  "gender": "female",
  "birthDate": "1990-05-15",
  "address": [
    {
      "use": "home",
      "line": ["123 Kamuzu Procession Road"],
      "city": "Lilongwe",
      "state": "Central Region",
      "postalCode": "P.O. Box 30377",
      "country": "MW"
    }
  ],
  "extension": [
    {
      "url": "http://dhis2.org/fhir/StructureDefinition/dhis2-org-unit",
      "valueReference": {
        "reference": "Organization/OrganizationFacilityA"
      }
    }
  ]
}
```

Key observations about the generated JSON:

- The `dhis2uid` slice appears as the first identifier with type code `RI`
- The `national` slice appears as the second identifier with type code `NI`
- The org unit extension uses the canonical URL from the extension definition
- All required demographics (name, gender, birthDate) are present
- The profile canonical URL appears in `meta.profile`
